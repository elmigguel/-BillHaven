```jsx
<form onSubmit={handleSignup}>

    />
    <label htmlFor="acceptTerms" className="text-sm">

      and not a financial or tax advisor.

  </div>

  <button type="submit" disabled={!acceptedTerms}>
    Create Account
  </button>
</form>
```

## QUICK REFERENCE: DO's and DON'Ts

### DO ✅
- Provide professional documentation
- State that documentation "may be useful for tax purposes"
- Encourage users to consult tax professionals
- Focus on cash flow benefits
- Offer transaction export tools
- Facilitate accountant access

### DON'T ❌
- Calculate tax savings for users
- Recommend specific deductions
- Act as tax advisor
- Guarantee tax benefits
- Create "what-if" tax calculators
- Answer "Can I deduct this?" questions
